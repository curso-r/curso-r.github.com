Estruturas de dados no R

Tipos bsicos de estrutura no R:

  • Atomic vector: homogneo e unidimensional
  • Matriz: homogneo e bidimensional
  • Array: homogneo e multidimensional
  • Lista: heterogneo e unidimensional
  • Data frame: heterogneo bidimensional

Nota: em sua implementao, atomic vectors e matrizes so tambm arrays e data frames so listas.

Atomic Vectors

Atomic vectors so a estrutura de objetos mais simples do R, caracterizados por no terem dimenso.

Tipos de atomic vectos:

  • lgico
  • integer
  • double
  • complexo
  • character

Exemplos:

dbl_var <- c(1, 2.5, 4.5) #DOUBLE
# Com o sufixo L temos nmeros inteiros em vez de double
int_var <- c(1L, 6L, 10L)
# Use TRUE ou FALSE (T ou F) para vetores lgicos
log_var <- c(TRUE, FALSE, T, F)
chr_var <- c("essas so", "algumas strings")

Curiosidade: na funo c(), o c de concatenate.

Para saber qual o tipo de um objeto, utilizamos a funo typeof().

typeof(dbl_var)
## [1] "double"
typeof(int_var)
## [1] "integer"
typeof(log_var)
## [1] "logical"
typeof(chr_var)
## [1] "character"

As funes is.integer(), is.double(), is.logical(), is.character() so usadas para testar se um objeto de um determinado tipo.

is.integer(dbl_var)
## [1] FALSE
is.double(dbl_var)
## [1] TRUE
is.numeric(dbl_var)
## [1] TRUE
is.logical(log_var)
## [1] TRUE
is.character(chr_var)
## [1] TRUE

Note que a funo is.numeric() retorna TRUE tanto para objetos double quanto para integer.

Coero

Quando dois tipos de objetos so inseridos uma estrutura homognea (atomic vectors, arrays ou matrizes), o R converte converter o objeto para o tipo mais flexvel, na ordem:

  • logical
  • integer
  • double
  • character

Na lista acima, character o tipo mais flexvel.

c("a", 1)
## [1] "a" "1"
c(T, 1)
## [1] 1 1

Isso pode ser til, por exemplo, para contar o nmero de TRUEs em um vetor lgico:

sum(c(T, F, T, F, T))
## [1] 3

Factors

Factors so utilizados para armazernar dados categorizados e so caracterizados por:

  • conterem apenas valores pr-definidos, chamados levels; e
  • se basearem num vetor de inteiros.
f <- factor(c("aventura", "terror", "comdia", "drama"))
f
## [1] aventura terror   comdia  drama   
## Levels: aventura comdia drama terror
levels(f)
## [1] "aventura" "comdia"  "drama"    "terror"

Observe que, dentro do objeto, os levels so organizados em ordem alfabtica.

Sempre tome cuidado ao converter factors em objetos numricos:

f <- factor(c("2", "3", "1", "10"))
as.numeric(f) # no funciona
## [1] 3 4 1 2
as.numeric(as.character(f)) # funciona
## [1]  2  3  1 10

Matrizes e Arrays

Matrizes e arrays so definidos usando as funes matrix() e array().

# Um vetor para descrever todas as dimenses
arr <- array(1:12, c(3,2,2))

# Dois argumentos para determinar o nmero de linahs e colunas
mat <- matrix(1:6, ncol = 3, nrow = 2)

Diferentemente dos atomic vectors, essas estruturas apresentam o atribuito dimenso.

Nota: observe que uma matriz um array com duas dimenses.

As funes length(), dim(), nrow(), ncol() so usadas para determinar o comprimento de cada dimenso de um objeto.

dim(c(1,2,3))
## NULL
nrow(c(1,2,3))
## NULL
ncol(c(1,2,3))
## NULL
length(c(1,2,3))
## [1] 3
dim(arr)
## [1] 3 2 2
length(arr)
## [1] 12
nrow(arr)
## [1] 3
ncol(arr)
## [1] 2
dim(mat)
## [1] 2 3
length(mat)
## [1] 6
nrow(mat)
## [1] 2
ncol(mat)
## [1] 3

Listas e Data frames

Listas so definidas usando a funo list().

list(
  c(1:5),
  c("homem", "mulher"),
  c(T, F, T),
  list(c(1,2,3), c("a", "b", "c"))
  )
## [[1]]
## [1] 1 2 3 4 5
## 
## [[2]]
## [1] "homem"  "mulher"
## 
## [[3]]
## [1]  TRUE FALSE  TRUE
## 
## [[4]]
## [[4]][[1]]
## [1] 1 2 3
## 
## [[4]][[2]]
## [1] "a" "b" "c"

Data frames so listas em que todos os elementos tm o mesmo comprimento. So definidos usando a funo data.frame().

df <- data.frame(x = 1:4, y = c("oi", "oi", "oi", "oi"), z = T)
df
##   x  y    z
## 1 1 oi TRUE
## 2 2 oi TRUE
## 3 3 oi TRUE
## 4 4 oi TRUE
str(df)
## 'data.frame':    4 obs. of  3 variables:
##  $ x: int  1 2 3 4
##  $ y: Factor w/ 1 level "oi": 1 1 1 1
##  $ z: logi  TRUE TRUE TRUE TRUE

Em um data frame converte, os caracteres so convertidos em factors. Se essa converso indesejvel, use o argumento stringAsFactors = F

df <- data.frame(x = 1:4, y = c("oi", "oi", "oi", "oi"), z = T, stringsAsFactors = F)
str(df)
## 'data.frame':    4 obs. of  3 variables:
##  $ x: int  1 2 3 4
##  $ y: chr  "oi" "oi" "oi" "oi"
##  $ z: logi  TRUE TRUE TRUE TRUE

Se usarmos a funo names() obtemos o nome das colunas do data frame. Tambm possvel mudar o nome das colunas:

names(df)
## [1] "x" "y" "z"
names(df) <- c("a", "b", "c")
names(df)
## [1] "a" "b" "c"

Combinando data frames

possvel combinar data frames usando as funes rbind() e cbind():

df1 <- data.frame(x = 1:4, y = c("s", "s", "s", "s"), z = T)
df2 <- data.frame(x = 1:2, y = c("n", "n"), z = F)

rbind(df1, df2)
##   x y     z
## 1 1 s  TRUE
## 2 2 s  TRUE
## 3 3 s  TRUE
## 4 4 s  TRUE
## 5 1 n FALSE
## 6 2 n FALSE
cbind(df1, df2)
##   x y    z x y     z
## 1 1 s TRUE 1 n FALSE
## 2 2 s TRUE 2 n FALSE
## 3 3 s TRUE 1 n FALSE
## 4 4 s TRUE 2 n FALSE

Subsetting no R

Chamamos de subsetting a seleo de um subconjunto de um objeto. No R, existem trs tipos principais de operao de subsetting:

  1. Nmeros inteiros positivos:

Retorna os elementos do vetor associados aos ndices especificados.

x <- c(13, 8, 5, 3, 2, 1, 1) 
x[c(1,2,3)] # seleciona os trs primeiros elementos do vetor
## [1] 13  8  5
order(x) # devolve a ordem dos elementos do vetor
## [1] 6 7 5 4 3 2 1
x[order(x)] # seleciona os elementos do vetor em ordem crescente
## [1]  1  1  2  3  5  8 13
  1. Nmeros inteiros negativos.

Exclui o elemento do vetor pelo ndice selecionado.

x[-c(2, 5,6)]
## [1] 13  5  3  1
  1. Vetores lgicos.

TRUE seleciona, FALSE no seleciona.

x == 1
## [1] FALSE FALSE FALSE FALSE FALSE  TRUE  TRUE
x[x==1]
## [1] 1 1
x > 5
## [1]  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE
x[x>5]
## [1] 13  8

Existem outros tipos de subsetting pouco (ou nada) utilizados:

x[] # retorna o prprio objeto
## [1] 13  8  5  3  2  1  1
x[0] # retorna um objeto de tamanho 0
## numeric(0)

Quando o objeto tem mais de uma dimenso, utilizamos a , para selecionar valores dentro de cada dimenso.

m <- matrix(c(1:5, 11:15), nrow=5, ncol=2)
m
##      [,1] [,2]
## [1,]    1   11
## [2,]    2   12
## [3,]    3   13
## [4,]    4   14
## [5,]    5   15
m[5,2] # Retorna o elemento especificado pelos ndices.
## [1] 15
m[,1] # Retorna todos os elementos da coluna 1.
## [1] 1 2 3 4 5
m[3,] # Retorna todos os elementos da linha 3.
## [1]  3 13
m[m%%2 == 0] # Retorna os elementos pares.
## [1]  2  4 12 14

Grficos com a funo plot()

ESCREVER INTRO

x <- seq(1, 10, 0.1)
y <- exp(-x)

plot(x, y)

Observe que o grfico gerado mapeia cada valor (x,y) como um ponto no plano cartesiano. Para mudar a forma de visualizao, utilizamos o argumento type=. Aqui esto os principais tipos de visualizao disponveis:

  • p para pontos (default)
  • l para retas
  • b para ambos (pontos e retas)
  • h para retas verticais
  • s para escadas
  • n para no plotar
plot(x, y, type = "l")

plot(x, y, type = "b")

plot(x, y, type = "h")

plot(x, y, type = "s")

plot(x, y, type = "n")

Para alterar a espessura das visualizaes, utilizamos o argumento lwd=:

plot(x, y, type = "p", lwd = 2)

plot(x, y, type = "h", lwd = 3)

Observe que esse argumento altera apenas a espessura da circunferncia do ponto. Para alterar o tamanho do ponto, utilizamos o argumento cex=:

plot(x, y, type = "p", lwd = 2, cex= 2)

Para alterar a cor do grfico, utilizamos o argumento col=:

plot(x, y, type = "h", lwd = 3, col = "red")

plot(x, y, type = "h", lwd = 3, col = "#9ff115")

O pacote graphics tambm traz funes para outros tipos de grficos mais especficos:

  • hist() - para histogramas
  • pie() - para grficos de pizza
  • boxplot() - para boxplots
