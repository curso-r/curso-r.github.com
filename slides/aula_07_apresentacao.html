Regresso Linear e ANOVA

summary()

A funo summary() uma funo genrica que geralmente devolve um resumo de informaes teis de praticamente qualquer classe de objetos. Para objetos lm() ela devolve:

  1. Chamada do modelo
  2. Medidas resumo dos resduos
  3. Tabela de coeficientes, desvios padro e testes T para hiptese nula de parmetros iguais a zero.
  4. Mdia dos quadrados do resduo e os respectivos graus de liberdade; \(R^2\) e \(R^2\) ajustado da regresso; Estatstica F para qualidade do ajuste (comparao com o modelo com apenas o intercepto).

Regresso Linear e ANOVA

stepwise

Stepwise no R se faz com as funes step() do pacote stats ou stepAIC() do pacote MASS. Escolher entre forward, backward ou both (ambos) basta passar um desses nomes ao parmetro direction da funo.

# modelo aditivo completo
ajuste_lm_completo <- lm(mpg ~ ., data = mtcars)

# modelo forward
step(ajuste_lm_completo, direction = "forward")

# modelo backward
step(ajuste_lm_completo, direction = "backward")

# modelo both
step(ajuste_lm_completo, direction = "both")

Regresso Linear Generalizada

MLG no R igualmente fcil. Mudamos de lm() para glm() e acrescentamos a famlia de distribuies que queremos assumir para a resposta.

Situao:

  • Suposio de normalidade no se verifica; e/ou
  • Homocedasticidade no se sustenta

Alternativa: regresso logstica/Poisson/Gama…

Regresso Linear Generalizada

Modelo

\[ Y_i \sim F(\mu_i, \phi) \]

\[ \mu_i = g^{-1}(\alpha + \beta_1 x_{i1} + \beta_p x_{ip}) \]

Regresso Linear Generalizada

Estrutura

ajuste_glm <- glm(resposta ~ explicativas, 
                  data = dados, 
                  family = distribuicao)

Para ver a lista de distribuies que podem ser passadas ao parmetro family, rode ?family no R.

Regresso Linear Generalizada

Funo de ligao

Outro componente importante em modelos lineares generalizados a funo de ligao. De modo mais geral, o cdigo para ajsutar um MLG fica assim:

ajuste_glm <- glm(resposta ~ explicativas, 
                  data = dados, 
                  family = distribuicao(link = funcao_de_ligacao))

Repare na parte (link = funcao_de_ligacao) depois do nome da distribuio escolhida.

Regresso Linear Generalizada

Funo de ligao

Uma prtica comum trocar a ligao inversa de uma regresso Gama para uma ligao log, por exemplo.

No R, fica:

ajuste_gama <- glm(Y ~ X + I(X^2) + Z, 
                   data = dados, 
                   family = Gamma(link = "log"))

OBS Todas as funes teis para lm() continuam sendo teis para glm().

rvore de Deciso

Cross-validation

  • arvores tendem a "super-ajustar" (overfit) o modelo.
  • Para evitar este problema, cross-validation uma boa pratica.
  • Cross-validation consiste em reservar parte da base reservada para depois ser usada para calcular o erro preditivo do modelo.
  • O erro preditivo vira o novo critrio de parada para a "podagem" da arvore.

rvore de Deciso

Cross-validation

Para fazer isso muito facil com a funo cv.tree(). Basta passar seu modelo ajustado para ela:

set.seed(123)
cv_tree <- cv.tree(ajuste_tree)
plot(cv_tree)

rvore de Deciso

Parmetros de controle

  • arvores de deciso so ajustadas sob alguns critrios que s vezes precisamos reconfigurar.
  • O ajuste_tree tinha muito mais galho do que o necessario. Isso se deveu a critrios de diviso/parada desregulados.
  • Objetos tree dois parmetros importantes: split e control.

rvore de Deciso

split

O parmetro split define qual o critrio para decidir se divide o "galho" ou no. Por padro, a funo utiliza o critrio "deviance", que tem a mesma definio do MLG. Uma segunda opo seria Gini.

control

O parmetro control recebe um objeto retornado pela funo tree.control().

rvore de Deciso

control

tree.control() permite voc configurar:

  • nobs Nmero de observaes na base de treino. Isso sera efetivo no cv.tree(), quando uma base de treino e outra de teste utilizada para calcular o erro de predio por meio de cross-validation.
  • mincut Nmero mnimo de observaes a serem includas em cada n.
  • minsize O menor tamanho de n permitido.
  • mindev Frao mnima do deviance do n raiz.

rvore de Deciso

control

Como passar funo:

controles <- tree.control(nobs = 32, mincut = 10)
ajuste_tree <- ajuste_tree <- tree(factor(am) ~ wt, 
                                   data = mtcars, 
                                   control = controles)
plot(ajuste_tree);text(ajuste_tree, pretty = 0)
